# Synthesized Concept

The bridge between episodic memory and curated knowledge. Machine-generated hypotheses about patterns emerging from reinforced episodic memories.

## Why a distinct type

A synthesized concept is **not** episodic (it has no single event time; it's an abstraction over many). It is **not** curated either (it hasn't been human-reviewed and isn't authoritative). Treating it as a third type lets the scorer weight it appropriately (between episodic and curated) and lets promotion be a first-class transition rather than a magic state flip.

See [[13-decisions/0002-planes-not-tiers]] for the full rationale.

## Pydantic model

```python
# musubi/types/concept.py

class SynthesizedConcept(BaseModel):
    object_id: KSUID
    namespace: str
    schema_version: int = 1

    title: str                          # generated by LLM, 3-10 words
    content: str                        # generated summary, 100-500 words
    synthesis_rationale: str            # LLM-generated: "clustered from memories X, Y, Z because..."
    tags: list[str] = Field(default_factory=list)
    topics: list[str] = Field(default_factory=list)

    importance: int = Field(ge=1, le=10)
    reinforcement_count: int = 0
    last_reinforced_at: datetime | None = None
    access_count: int = 0
    last_accessed_at: datetime | None = None

    # Temporal
    created_at: datetime
    created_epoch: float
    updated_at: datetime
    updated_epoch: float

    # Lifecycle
    version: int = 1
    state: LifecycleState                # see allowed transitions below

    # Lineage (required for concepts — this is the whole point)
    merged_from: list[KSUID] = Field(min_length=3)  # at least 3 source memories
    merged_from_planes: list[str] = Field(default_factory=list)  # which planes contributed

    # Promotion
    promoted_to: KSUID | None = None            # the CuratedKnowledge that was promoted
    promoted_at: datetime | None = None
    promotion_rejected_at: datetime | None = None
    promotion_rejected_reason: str | None = None
    promotion_attempts: int = 0

    # Relationships
    supersedes: list[KSUID] = Field(default_factory=list)
    superseded_by: KSUID | None = None
    contradicts: list[KSUID] = Field(default_factory=list)
    linked_to_topics: list[str] = Field(default_factory=list)
    supported_by: list[ArtifactRef] = Field(default_factory=list)
```

## Allowed lifecycle states

For concepts specifically:

```
synthesized → matured → promoted      (success path)
             ↘
              → demoted              (low reinforcement over time; not promotable)
             ↘
              → archived             (explicit archival; keeps lineage)
```

Transitions:

- `synthesized` (on creation from synthesis job).
- `synthesized → matured` (24h after creation, if no contradiction surfaced).
- `matured → promoted` (promotion gate passes; CuratedKnowledge written).
- `matured → demoted` (decay rule; no reinforcement in 30 days).
- `* → archived` (operator action).
- `* → superseded` (a newer concept merges in and replaces).

## Qdrant layout

Collection: `musubi_concept`.

Indexes: standard (`namespace`, `object_id`, `state`, `tags`, `topics`, `created_epoch`, `updated_epoch`, `importance`, `reinforcement_count`, `access_count`) + concept-specific:

| Field | Type | Purpose |
|---|---|---|
| `promoted_to` | KEYWORD | reverse lookup from curated to source concept |
| `promotion_attempts` | INTEGER | failure-retry analytics |
| `merged_from_planes` | KEYWORD | "what mix of sources?" filter |

## Synthesis input selection

The synthesis job (see [[06-ingestion/concept-synthesis]]) produces concepts from *clusters of matured episodic memories*. Input selection:

1. Scroll `musubi_episodic` for all memories in this namespace where `state = matured` AND `updated_epoch > last_synthesis_run`.
2. Cluster by:
   - Dense vector similarity (HDBSCAN with min_cluster_size=3, or simpler: threshold-cluster at 0.80).
   - OR shared tags/topics.
3. For each cluster of ≥ 3 memories: send to LLM ("what common theme? What salient fact? Write title + 300-word summary").
4. Check against existing concepts (same namespace, dense similarity ≥ 0.85). If match: *reinforce* existing concept, don't create new.

This is the **Mem0 extract-consolidate pattern** applied at the concept layer.

## Promotion gate

A concept is eligible for promotion when all are true:

- `reinforcement_count ≥ 3`
- `importance ≥ 6`
- `created_at` more than 48 hours ago (let contradictions surface)
- No active `contradicts` entries of higher `provenance_strength`
- `state` is `matured` (not synthesized; not demoted)
- `promotion_attempts < 3` (avoids infinite retry on broken concepts)

Gate tunables live in `config.py` as `PROMOTION_REINFORCEMENT_THRESHOLD`, etc. All per-namespace configurable in the future; for v1, global.

## Reinforcement

A concept is reinforced when:

1. **Synthesis job finds a new matured memory that semantically matches** (dense ≥ 0.85 to concept vector).
2. **Episodic retrieval surfaces it** (recalling a concept-linked memory counts as weak reinforcement; bumps `access_count`, not `reinforcement_count`).

Distinguishing these matters: we want promotion to be driven by *new evidence*, not just re-reads of existing evidence.

## Contradiction detection

During synthesis, when a candidate concept overlaps semantically with an existing one (≥ 0.75 but < 0.85), the LLM is asked: "Are these consistent, or do they contradict?" If contradicting, both concepts get `contradicts` entries pointing at each other. Both are blocked from promotion until a human resolves via the CLI:

```bash
musubi-cli contradictions resolve <concept-a> <concept-b> --keep=a --reason "B was about a deprecated setup"
```

## Prior art

- Mem0 ADD/UPDATE/DELETE/NOOP: [https://arxiv.org/abs/2504.19413](https://arxiv.org/abs/2504.19413) — our synthesis mirrors the ADD/UPDATE paths; DELETE is our demotion; NOOP is the "below threshold" skip.
- Generative Agents "reflection" pass: [https://arxiv.org/abs/2304.03442](https://arxiv.org/abs/2304.03442) — we differ by separating reflection (daily summary; see [[06-ingestion/reflection]]) from synthesis (concept creation).

## Open questions

- Should we let humans create concepts directly? Current answer: **no.** If a human wants to state a concept, they write a CuratedKnowledge file. Concepts are machine-generated.
- Should concepts be visible to ordinary queries? Current answer: yes, but with a provenance weight lower than curated. See [[05-retrieval/scoring-model]].
